# 1. 서비스 개요

## 1.1 서비스 목적

InFit은 뷰티 브랜드 마케터가 채팅 인터페이스를 통해 브랜드 요구사항을 입력하면, AI Agent가 YouTube 인플루언서를 자동 탐색·평가하여 Top 10 적합 인플루언서를 순위로 추천하는 서비스다. 사용자는 AI의 질문에 답하는 방식으로 조건을 제공하며, AI는 정량 지표·이미지 품질·텍스트 품질·리스크를 통합 채점한 결과를 채팅 내에서 즉시 제공한다.

## 1.2 타겟 사용자

| 구분 | 대상 | 핵심 니즈 |
|------|------|-----------|
| Primary | 뷰티 브랜드 마케팅/PR 담당자 | 인플루언서 선정 시간 단축, 객관적 근거 확보 |
| Secondary | 인플루언서 마케팅 대행사 담당자 | 다수 후보 빠른 스크리닝 |

## 1.3 핵심 가치 제안

- 채팅으로 조건 입력 → AI 자동 탐색 → Top 10 순위 추천까지 단일 플로우
- 저장/회원가입 없이 즉시 사용
- 정량(구독자·조회수·인게이지먼트) + 콘텐츠 이미지 품질(Gemini 멀티모달) + 텍스트 품질 + 리스크 통합 채점
- 뷰티 도메인 특화 — B&A 표현력·시각적 완성도·브랜드 톤 가중치 반영

# 2. 기술 스택

| 구분 | 기술 | 선택 이유 |
|------|------|-----------|
| 프레임워크 | React 18 + Vite + TypeScript | 빠른 빌드, Vercel 최적화, 타입 안전성 |
| 스타일 | SCSS (CSS Modules) | 컴포넌트 스코프 스타일링, 변수/믹스인 활용 |

| 차트 | Recharts | 결과 카드 내 점수 막대 시각화 |
| AI | Gemini API (gemini-1.5-flash) | YouTube 썸네일 멀티모달 분석 + 텍스트 평가 |
| 데이터 | YouTube Data API v3 | 채널 탐색·정량 지표·영상 정보 수집 |
| 배포 | Vercel | 환경변수 관리, 자동 CI/CD |

# 3. 서비스 플로우

## 3.1 전체 사용자 플로우

```
[랜딩 페이지] → "분석 시작하기" 클릭
    ↓
[채팅 인터페이스 진입]
    ↓
AI: "홍보할 제품명을 알려주세요."
    ↓ 사용자 입력
AI: "제품의 핵심 특징을 입력해주세요."
    ↓ 사용자 입력 (예: 지속성, 발색력)
AI: "타겟층을 알려주세요."
    ↓ 사용자 입력 (예: 미국 10대 학생들)
AI: "검색 키워드를 선택해주세요."
    ↓ 사용자 입력 (예: glow)
AI: "구독자 범위를 선택해주세요." [1만~10만] [10만~50만] [50만~]
    ↓ 사용자 선택
AI: "입력 정보를 확인했습니다." (요약 카드 표시)
    ↓
[분석 진행] — 5단계 체크리스트 순차 표시
  ✓ 인플루언서 탐색 중...
  ✓ 콘텐츠 이미지 분석 중...
  ✓ 텍스트 적합성 평가 중...
  ✓ 리스크 스크리닝 중...
  ✓ 최종 점수 산출 중...
    ↓
[결과] — Top 10 순위 리스트 (추천/비추천 분류)
```

| 3 | "타겟층을 알려주세요." | 텍스트 입력 | 미국 10대 학생들 |

## 4.4 결과 카드 상세

### 추천 채널 카드 (확장형 — 1위)
- 채널명 + 점수 (84.2) + 구독자 수
- 배지: 추천 (초록) / 비추천 (빨강)
- 점수 막대 3개: 정량 / 이미지 / 텍스트
- 추천 이유 텍스트

### 비추천 채널 카드
- 채널명 + 점수 + 구독자 수
- 배지: 비추천 (빨강)
- 점수 막대 3개
- 비추천 이유 플래그 태그 (예: "높은 FSI", "리스크 감지", "낮은 이미지 품질")

### 결과 요약 헤더
- 분석 채널 수 / 추천 채널 수 / 최고 점수 3개 숫자

# 5. 평가 파이프라인

## 5.1 자동 채널 탐색

```
사용자 조건 (키워드 + 구독자 범위)
    ↓
YouTube Search API → 키워드로 채널 검색 (최대 20개)
    ↓
구독자 범위 필터링
    ↓
분석 대상 채널 확정 (최대 10~20개)
```

## 5.2 평가 파이프라인 (채널당)

| 단계 | 내용 | 출력 |
|------|------|------|
| Step 1 | 정량 데이터 수집 | 구독자·조회수·인게이지먼트율·업로드 빈도 |
| Step 2 | 허수 의심 지수 (FSI) | FSI 0~1 (0.6 이상 감점) |
| Step 3 | 이미지 평가 (Gemini 멀티모달) | 썸네일 3장 → 시각 완성도·B&A·브랜드톤·제품노출 1~5점 |
| Step 4 | 텍스트 평가 (Gemini) | 제목·설명·태그 → 소비자공감·신뢰성·CTA·해시태그 1~5점 |
| Step 5 | 리스크 스크리닝 (Gemini) | 리스크 플래그 없음/낮음/높음 |
| Step 6 | 통합 채점 | 최종 적합도 점수 0~100 |

## 5.3 채점 공식 (뷰티 도메인)

```
최종 점수 = (정량 점수 × 0.30) + (이미지 점수 × 0.40) + (텍스트 점수 × 0.30)

페널티:
- FSI ≥ 0.6 → -15점
- 리스크 플래그 '높음' → -10점
```

## 5.4 이미지 평가 항목 (Gemini 프롬프트 기준)

| 평가 항목 | 설명 | 점수 |
|-----------|------|------|
| 시각적 완성도 | 구도·조명·편집 품질 | 1~5 |
| B&A 표현력 | Before & After 구성 및 커버력 변화 시각 표현 | 1~5 |
| 브랜드 톤 적합성 | 브랜드 컬러·무드와의 일치 여부 | 1~5 |
| 제품 노출 자연스러움 | 광고티 없이 제품이 자연스럽게 등장하는지 | 1~5 |

## 5.5 텍스트 평가 항목 (Gemini 프롬프트 기준)

| 평가 항목 | 설명 | 점수 |
|-----------|------|------|
| 소비자 공감력 | 일상 구어체·공감 유발 표현 비율 | 1~5 |
| 신뢰성 표현 | 과장·허위 없는 자연스러운 후기 여부 | 1~5 |
| CTA 자연스러움 | 광고성 vs 자연스러운 추천 톤 분석 | 1~5 |
| 해시태그 적절성 | 트렌드 및 뷰티 관련 해시태그 사용 여부 | 1~5 |

# 6. API 설계 (Vercel Serverless Functions)

| 엔드포인트 | 메서드 | 역할 | 사용 API |
|-----------|--------|------|----------|

| `/api/risk-screen` | POST | 리스크 키워드 스크리닝 | Gemini API (gemini-1.5-flash) |
